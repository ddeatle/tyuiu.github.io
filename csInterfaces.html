Интерфейсы и абстрактные классы

Интерфейсы

Интерфейсы определяют контракт, который должны реализовывать классы. Интерфейс содержит только декларацию методов, но не их реализацию.

 1  interface IFlyable
 2 {
 3      void Fly();
 4 }
 5 
 6  class Eagle : Animal, IFlyable
 7 {
 8      public override void MakeSound()
 9      {
10         Console.WriteLine("Кар-кар!");
11      }
12 
13      public void Fly()
14      {
15         Console.WriteLine("Орел летит");
16      }
17 }

Абстрактные классы

Абстрактные классы содержат как декларации методов, так и их реализацию. Они используются для объединения общих свойств и методов в базовом классе.

 1  abstract class FlyingAnimal : Animal
 2 {
 3      public abstract void Fly();
 4      public override void MakeSound()
 5      {
 6          Console.WriteLine("Животное издает звук во время полета");
 7      }
 8 }
 9 
10  class Sparrow : FlyingAnimal
11 {
12      public override void Fly()
13      {
14          Console.WriteLine("Воробей летит");
15      }
16 }

В этом примере интерфейс IFlyable определяет контракт для летающих животных, который реализует класс Eagle. Абстрактный класс FlyingAnimal содержит общую реализацию метода MakeSound() и объявление абстрактного метода Fly(), который реализуется в конкретном классе Sparrow.
